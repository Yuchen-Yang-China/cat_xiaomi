# 小咪猫咪项目

这是一个用“管理/开发项目”的方式维护小猫生活、健康、行为、训练和知识学习的本地知识库。目标是让你以后无论在哪里打开这个仓库，都能把它作为长期上下文，向本地智能体提问、补充记录、复盘决策。

## 建议使用方式

1. 每次和智能体讨论小猫问题时，先让智能体阅读 `AGENTS.md`、`INDEX.md` 和相关目录。
2. 日常发生的事情先写进 `00_inbox/` 或对应模板，再整理到长期文件。
3. 涉及症状、用药、疫苗、驱虫、绝育、急救等内容时，把事实记录清楚，但医疗判断以兽医为准。
4. 所有重要决定都写入 `08_questions_decisions/decisions.md`，避免以后忘记当时为什么这样做。
5. 定期提交到 GitHub：小改动勤提交，大调整写清楚提交信息。

## 目录总览

- `00_inbox/`：临时收集区，任何还没分类的信息先放这里。
- `01_profile/`：小猫身份、成长时间线、偏好、基线状态。
- `02_daily/`：日常日志、作息、睡眠活动等。
- `03_health/`：体重、疫苗、驱虫、就诊、症状、用药、应急预案。
- `04_nutrition/`：主食、零食、饮水、喂食记录和反应。
- `05_behavior_training/`：行为观察、训练计划、猫砂盆、抓挠、咬人、压力源、社会化。
- `06_environment_supplies/`：家居布置、用品清单、清洁安全、玩具丰容、梳毛护理。
- `07_knowledge/`：你学习到的猫咪行为、心理、健康、营养、安全知识。
- `08_questions_decisions/`：问题、AI 回答、实验记录、最终决策。
- `09_media/`：照片、视频、体检单、处方、票据等附件。
- `10_templates/`：常用记录模板。

## 首次需要补充的信息

从 `01_profile/cat_profile.md` 开始填写：

- 小猫姓名、昵称、性别、生日或估计年龄。
- 到家日期、来源、品种或外观特征。
- 当前体重、饮食、猫砂、疫苗、驱虫、绝育状态。
- 已知疾病、过敏、特殊行为、害怕或喜欢的东西。
- 常用医院、兽医联系方式、紧急联系人。

## GitHub 维护建议

- 强烈建议先创建 GitHub private repository，避免医院、地址、联系方式和附件被公开。
- 日志和知识优先用 Markdown，方便搜索和 diff。
- 大视频不建议直接进 Git；如果要保存大量媒体，后续可以考虑 Git LFS 或云盘链接。
- 每次同步前运行 `git status` 看是否有未提交改动。
- 推荐提交信息示例：
  - `docs: add initial cat profile`
  - `log: add 2026-07 daily notes`
  - `health: record vaccine visit`
  - `behavior: update biting observation`
