# 仓库优化路线图

这份路线图只记录“仓库怎样更好维护”，不混入小咪的照护待办。照护事项见 `08_questions_decisions/action_queue.md`。

## 已完成：第一阶段

- 明确原始记录、当前摘要、派生洞察和历史快照四层角色。
- 重做 `README.md`、`INDEX.md` 和 `project_manifest.yml`，提供面向公众、主人/智能体和程序的三层导航。
- 增加统一行动队列，避免疫苗、驱虫和复查计划散落在多个文件中。
- 增加 `11_insights/` 多维派生数据和综合报告。
- 增加可直接双击打开的 `dashboard.html`，并提供构建与一致性检查脚本。
- 补充更自然的回答模式和语气示例，减少固定模板感。
- 清理公开文件中的本机绝对路径，修正饮水基线和体重日期口径。

## 下一阶段：结构化原始数据

优先级：高。

目标是让图表和分析从结构化事实生成，而不是在 Markdown、JSON 和 HTML 中分别维护。

- 增加 `03_health/data/weight_measurements.csv`，保存时间、日龄、体重、日期精度、测量方法、来源和备注。
- 增加 `03_health/data/health_events.csv`，使用稳定事件 ID，并区分观察、是否兽医确诊、混杂因素、处理和结果。
- 增加 `05_behavior_training/data/behavior_events.csv`，保留 ABC、强度、尝试次数、结果和中断条件。
- 为日常数据建立长表，统一 `domain / metric / value / unit / window / value_type / source_path`。
- 让 `current_snapshot.json` 由上述数据自动生成，减少手工同步。

## 下一阶段：降低空模板噪音

优先级：中。

- 将未知状态区分为：重要待确认、尚未观察、历史不可补齐、因隐私不公开、不适用。
- 在 manifest 中给几乎为空的文件标记 `inactive`，默认回答时跳过。
- 把训练计划收缩为 1-2 个活跃目标；通用 SOP 移到知识或协议文件，完成/暂停项目归档。
- 把“今天的小片段、喜欢的互动仪式、放松信号”补进档案，让仓库不只记录风险和缺口。

## 已有基础、下一阶段继续扩展：自动检查

优先级：中。

- 当前已有 `python3 scripts/check_repository.py`，检查 Markdown 内链、CSV 格式、公开路径和看板同步。
- 下一步补 YAML 结构、重复事件 ID、日期、单位和枚举值校验。
- 检查未来计划是否过期、`as_of` 是否落后于最新日志、生成物是否未更新。
- 扫描绝对本机路径、疑似隐私字段和媒体 EXIF。
- 数据结构稳定后，再考虑 pre-commit 或 GitHub Actions；公开推送仍需主人确认。

## 长期可选项

优先级：低。

- 为 `09_media/` 增加附件清单，记录用途、日期、脱敏状态、EXIF 处理和关联事件。
- 使用稳定图表文件名，避免文件名日期和图中数据更新日期漂移。
- 决定模板、脚本和小咪个人档案是否需要不同许可；当前继续保持无 LICENSE 也可以。
- 数据积累到至少连续 3-7 天后，再增加滚动均值、波动区间和行为训练成功率，不提前制作缺少分母的评分。
