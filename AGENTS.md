# 智能体工作说明

这个仓库是“小咪”的猫咪生活档案和知识库。任何智能体在回答问题或修改文件前，应优先读取本文件、`INDEX.md`、`project_manifest.yml`，再根据问题读取相关目录。

## 回答原则

- 先区分“已记录事实”和“推测/建议”。不要把没有记录的内容当成事实。
- 涉及健康、用药、疫苗、驱虫、急救、绝育、异常症状时，必须提醒以兽医诊断为准。
- 如果信息不足，先列出需要补充的关键事实；能基于现有资料给出低风险建议时，可以明确标注假设。
- 回答时尽量引用本仓库中的具体文件路径，方便追溯。
- 新增信息时优先追加到已有文件，不要随意创建重复目录。

## 推荐阅读顺序

1. `01_profile/cat_profile.md`
2. `01_profile/baseline.md`
3. 最近的 `02_daily/logs/` 日志
4. 与问题相关的专项目录：
   - 健康：`03_health/`
   - 饮食：`04_nutrition/`
   - 行为训练：`05_behavior_training/`
   - 环境用品：`06_environment_supplies/`
   - 已学习知识：`07_knowledge/`
   - 历史问题和决策：`08_questions_decisions/`

## 写入规则

- 事实记录使用日期，格式为 `YYYY-MM-DD`。
- 不确定信息使用 `待确认` 或 `估计` 标记。
- 观察和结论分开写：先写发生了什么，再写可能原因。
- 医疗相关记录保留原始凭证或照片路径，并写明医院、兽医、剂量、时间。
- 重要决定写到 `08_questions_decisions/decisions.md`。
